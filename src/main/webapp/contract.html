What We Did Together


We reviewed the code that each of us made together to ensure that everyone understood what had been made, and we made sure that everyone was present when we did pull requests on GitHub (merging onto the master branch).
We also all helped out whenever someone ran into problems with their parts, and came up with solutions to problems together. We spent most of the time together in a Discord chat room, even when working on our separate parts, so it was easy to help out when someone ran into problems.

Group Contract


1. Every new feature must be made on its own branch

2. No changes are made directly on the master branch except bug fixes

3. Everyone must review changes before a pull request is made (merged to master)

4. If anyone runs into problems the entire group should be willing to help solve the problem
